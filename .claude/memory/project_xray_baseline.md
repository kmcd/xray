xray v0.4.1 was tagged 2026-06-10 (same day as v0.4.0, which was the "first-customer polish" release). Single contributor (the user). Trunk-based, direct-to-main, admin-bypass on `main` for human commits.

**Load-bearing contracts (do not break casually):**

- `schema_version = 2`. Bumped from `1` at v0.3.0 (author-handle hashing). The downstream Ruby analyser refuses unknown values. This — not semver — is the artifact-compatibility contract. See [[feedback-schema-vs-semver]].
- Connector read-only invariant. No `POST`/`PATCH`/`PUT`/`DELETE` calls in any `internal/connectors/X`. Asserted at code review.
- "No source content" invariant. `commit_files` is numstat only; `file_metrics` is byte-scan stats; commit/PR bodies parsed and discarded; `harness_artifacts.content` empty unless `capture_harness_content = true`.
- Static binary / CGO-free. `modernc.org/sqlite` is the SQLite driver for this reason. Don't propose `mattn/go-sqlite3`.

**Install surface as of v0.4.1:**

- macOS: brew Cask via `brew tap kmcd/xray https://github.com/kmcd/xray && brew install kmcd/xray/xray`. Cask is `Casks/xray.rb` on `main`, generated by `scripts/publish-tap.sh` (templated; includes a `postflight` block that strips `com.apple.quarantine` because the binary is cosign/Sigstore-signed but not Apple-notarized).
- Linux: `curl -sSfL .../scripts/install.sh | sh`. POSIX `sh`, sha256 verify against `checksums.txt`, opt-in cosign via `XRAY_VERIFY_COSIGN=1`.
- Windows: Scoop bucket via `bucket/xray.json` on `main` (untested locally; no Windows VM wired up).
- Manual: direct download from releases; the cosign-verify recipe is in README under "Verifying the binary".

**Release flow:** `/release vX.Y.Z` is one invocation. It pre-flights (gates + prose green, version valid, `[Unreleased]` non-empty), confirms an intro paragraph, rolls CHANGELOG + bumps README, commits, tags, pushes, watches CI, runs `scripts/publish-tap.sh`, smokes `brew install`. The `homebrew_casks:`/`scoops:` stanzas in `.goreleaser.yaml` carry `skip_upload: true` because the workflow bot can't bypass `main` BP on a personal repo — see [[reference-github-bot-bypass-personal-repo]].

**Why:** v0.4.x ships to consultants who run xray in client environments and review the binary against cosign + SLSA L3 + SBOM before doing so. The brew/scoop paths are a customer-trust improvement over the old `curl | tar` snippet; the supply-chain story (cosign + SLSA + SBOM) is preserved on all paths.

**How to apply:**
- Distinguish `schema_version` (analyser contract) from semver (binary contract). See [[feedback-schema-vs-semver]].
- When proposing release tooling changes, run the new shape end-to-end against a real release before declaring done. See [[feedback-release-tooling-smoke]].
- When discussing branch protection or bot-push, remember the personal-repo Ruleset constraint. See [[reference-github-bot-bypass-personal-repo]].
- `/release` is the release entry point. `/publish-tap` is the recovery path if `/release` fails between tag-push and tap-publish. `/ready` Step 6.5 warns if the latest tag's Cask + manifest haven't been published.
